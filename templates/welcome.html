Configuring Galaxy

On the Galaxy machine, add the following configuration to your Galaxy job_conf to enable the use of this Pulsar endpoint

YAML

Your job configuration is probably at /srv/galaxy/config/job_conf.yml

  1. Open /srv/galaxy/job_conf.yml with a text editor
  2. Find the section starting with runners: at the start of the line, it should be at the start of the file.
  3. Add a destination after the local: runner like this, it will say that the pulsar runner should use the PulsarRESTJobRunner module. 
      pulsar:
    load: galaxy.jobs.runners.pulsar:PulsarRESTJobRunner
  4. Finally it should look like: 
    runners:
  local:
    load: galaxy.jobs.runners.local:LocalJobRunner
    workers: 4
  pulsar:
    load: galaxy.jobs.runners.pulsar:PulsarRESTJobRunner

Next we will configure the Pulsar endpoint itself.

  1. Open /srv/galaxy/job_conf.yml with a text editor
  2. Find the section starting with execution: at the start of the line, it should be in the middle of the file.
  3. Add a destination after the local: runner like this, it will say that the pulsar runner should use the PulsarRESTJobRunner module. 
        remote_{{ ansible_hostname }}:
       runner: pulsar
       url: https://{{ workspace_fqdn | default('localhost', true)  }}
       private_token: {{ pulsar_token | default('unknown', true) }}
       dependency_resolution: remote
       manager: _default_
       # Uncomment the following to enable interactive tools:
       # docker_enabled: true
       # docker_set_user: null
       # docker_memory: "8G"
       # singularity_enabled: false
       # tmp_dir: true
       # outputs_to_working_directory: false
       # container_resolvers:
       # - type: explicit
       # require_container: True
       # container_monitor_command: /mnt/pulsar/venv/bin/galaxy-container-monitor
       # container_monitor_result: callback
       # container_monitor_get_ip_method: command:echo {{ workspace_fqdn | default('localhost', true) }}
  4. Finally it should look like: 
    execution:
  default: docker_dispatch
  environments:
    local:
      runner: local

    # Maybe some other runners are here.
    remote_{{ ansible_hostname }}:
       runner: pulsar
       url: https://{{ workspace_fqdn | default('localhost', true)  }}
       private_token: {{ pulsar_token | default('unknown', true) }}
       dependency_resolution: remote
       manager: _default_
       # Uncomment the following to enable interactive tools:
       # docker_enabled: true
       # docker_set_user: null
       # docker_memory: "8G"
       # singularity_enabled: false
       # tmp_dir: true
       # outputs_to_working_directory: false
       # container_resolvers:
       # - type: explicit
       # require_container: True
       # container_monitor_command: /mnt/pulsar/venv/bin/galaxy-container-monitor
       # container_monitor_result: callback
       # container_monitor_get_ip_method: command:echo {{ workspace_fqdn | default('localhost', true) }}
